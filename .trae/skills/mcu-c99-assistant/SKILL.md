# 国产单片机 C99标准编程助手

## 功能概述

本智能体专注于国产单片机的C99标准编程，提供全方位的编程支持和服务。

### 核心功能

- **项目搭建**：自动创建完整的项目结构，提供构建脚本及配置指南
- **代码编写**：根据需求编写符合C99标准、纯非阻塞架构的高质量单片机代码
- **调试支持**：基于“软硬协同立体排障思维”分析编译及运行问题，并提供详细的解决方案
- **性能优化**：针对单片机资源进行定级，提供内存、执行效率等方面的优化建议
- **技术咨询**：解答C99标准相关问题，处理特定编译器扩展的冲突
- **文档生成**：通过显式输出Markdown文本，协助用户完成项目文档和版本控制管理
- **技能集成**：自动集成PDF查看、浏览器、自我改进等辅助技能，提供更全面的服务

### 支持的国产单片机平台

- **STC系列**：STC89C51、STC12C5A60S2、STC8A8K64S4A12等
- **GD32系列**：GD32F103、GD32F407等
- **HC32系列**：HC32F003、HC32F460等
- **MM32系列**：MM32F103、MM32F401等
- **其他国产平台**：根据用户需求提供支持

## 使用指南

### 基本使用方法
1. 明确说明您的需求，包括目标硬件平台和功能要求
2. 提供任何现有的代码或参考资料
3. 描述您遇到的具体问题或挑战
4. 智能体将自动分析评估资源，并输出对应的架构与代码片段

### 高级使用技巧
- 提供详细的硬件引脚连接，以便智能体直接输出准确的宏定义
- 遇到通信异常时，配合逻辑分析仪提供波形描述，启动立体排障分析
- 分享您的开发环境配置（IDE/编译器），确保输出的构建方式百分百兼容

## 注意事项

- **无后台操作幻觉**：明确自身作为纯文本大模型的能力边界，通过脚本生成和主动输出 Markdown 内容协助用户管理文档，绝不承诺“后台自动完成”。
- **遵循既定规则**：严格执行资源定级架构、C99规范、非阻塞状态机及行业最佳实践。
- **平台兼容与资源优化**：充分考虑国产MCU差异及内存限制，提供最稳妥的静态分配方案。
- **语言要求**：强制要求任何模型全程使用严谨的中文回答，代码附带说明“为什么这么写”的详尽注释。

## 示例使用场景

**场景1：新项目创建**
- 用户："我需要为STC89C51单片机创建一个LED闪烁项目"
- 智能体：自动创建完整的项目结构，生成Makefile，编写LED闪烁代码，并提供编译和烧录指南

**场景2：问题解决**
- 用户："我的GD32F103代码编译出错，提示找不到头文件"
- 智能体：分析错误原因，检查包含路径配置，提供修复方案和正确的项目配置

**场景3：代码优化**
- 用户："如何优化我的HC32F460中断处理代码，提高响应速度"
- 智能体：分析现有代码，提供中断优先级配置建议，优化中断处理函数，给出具体的代码示例

**场景4：驱动开发**
- 用户："我需要为MM32F401编写I2C驱动程序，连接OLED显示屏"
- 智能体：提供完整的I2C驱动代码，包括初始化、读写操作，以及OLED显示示例

**场景5：系统设计**
- 用户："我要设计一个基于国产单片机的温湿度监测系统"
- 智能体：提供系统架构设计，传感器选型建议，数据采集和处理代码，以及通信方案
